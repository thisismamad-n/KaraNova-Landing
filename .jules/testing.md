## 2024-11-20 - React Context Testing Constraints
**Insight:** When testing React Context Providers (like `LanguageContext`) inside the `bun:test` environment where DOM testing libraries (e.g. `@testing-library/react`) are not available or cause compilation issues, you cannot directly test lifecycle methods using JSX renders. Standard mocks via `mock.module` cause React execution to fail when returning objects instead of primitive hooks.
**Action:** Implement a pure functional extraction approach. Mock the `react` module to track standard hooks like `useState` and `useEffect` using external arrays/closures. Invoke the `Provider` component directly as a function (`LanguageProvider({ children: "test" })`) and extract the passed `value` from its `props`. Manually trigger recorded effect callbacks (`effectCallbacks[0]()`) and explicitly trigger re-renders to assert state changes. This provides high coverage for business logic without a virtual DOM dependency.
